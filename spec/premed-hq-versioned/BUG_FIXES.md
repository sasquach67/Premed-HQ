# Bug Fixes

Use this file as the single intake queue for defects, regressions, visual problems, and small corrective changes.

## Workflow

1. Add each issue under **Open bugs** using the template below.
2. Give every issue a stable ID: `BUG-001`, `BUG-002`, and so on.
3. Before Codex edits code, create or confirm a version snapshot.
4. Ask Codex to fix only the listed bug IDs unless the request explicitly includes a feature change.
5. After testing, move resolved items to **Resolved bugs** and record the version.
6. Never delete prior entries; the file is also a lightweight defect history.

## Bug template

```md
### BUG-000 — Short title
- **Status:** Open
- **Found in version:** v0.0.0
- **Area:** Overview / Academics / Sidebar / etc.
- **Severity:** Low / Medium / High / Critical
- **Description:** What is visibly or functionally wrong.
- **Expected behavior:** What should happen instead.
- **Reproduction steps:**
  1. ...
  2. ...
- **Evidence:** Screenshot filename or notes.
- **Constraints:** Anything Codex must preserve.
- **Resolved in:** —
- **Resolution notes:** —
```
