This project uses a simplified semantic-versioning scheme:

- **PATCH** (`v0.1.1`) — bug fixes, copy changes, minor styling corrections
- **MINOR** (`v0.2.0`) — a new feature, page redesign, or meaningful UX change
- **MAJOR** (`v1.0.0`) — a large architectural migration, incompatible data change, or major product milestone
